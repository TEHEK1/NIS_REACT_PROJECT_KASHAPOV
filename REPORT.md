# NFT Gallery — Отчёт

## Суть приложения

**NFT Gallery** — это веб-приложение для просмотра реальных NFT-коллекций на блокчейне Ethereum. Все данные загружаются on-chain — никаких моков и фейковых данных. Приложение представляет собой SPA, где пользователь может:

- Просматривать NFT из реальных ERC-721 контрактов (BAYC, Azuki, Pudgy Penguins, Doodles, Cool Cats и любых других)
- Добавлять произвольные ERC-721 контракты в каталог по адресу — приложение валидирует контракт через вызов `name()` и автоматически подгружает токены
- Видеть метаданные NFT (имя, описание, изображение, свойства), загруженные через `tokenURI()` с резолвом IPFS/Arweave-ссылок
- Изучать информацию о контрактах (name, symbol, totalSupply) — всё из блокчейна
- Просматривать реальную историю Transfer-событий с ссылками на Etherscan
- Подключать MetaMask и смотреть баланс, сеть, номер блока, gas price
- Просматривать баланс и количество транзакций любого Ethereum-адреса
- Бесконечно скроллить коллекции — новые NFT подгружаются автоматически через IntersectionObserver
- Сохранять NFT в избранное, переключать тему, фильтровать и искать

Приложение работает без MetaMask — через публичный RPC-fallback (`ethereum-rpc.publicnode.com`).

## React

### Общее описание
React — это JavaScript-библиотека для построения пользовательских интерфейсов, разработанная Meta. Использует декларативный подход, компонентную архитектуру и виртуальный DOM.

### Зачем использовался
React является основой приложения. Все UI-элементы — React-компоненты с хуками. Ключевые паттерны:

- **React Context** — глобальное состояние (тема, кошелёк, toast-уведомления) без prop drilling
- **React.lazy + Suspense** — ленивая загрузка страниц (code splitting)
- **memo** — мемоизация компонентов (`NFTCard`, `CollectionCard`, `FilterPanel`) для предотвращения лишних перерисовок
- **useSyncExternalStore** — подписка на внешний ContractStore для реактивного обновления каталога контрактов
- **Custom hooks** — инкапсуляция логики: `useContracts`, `useContractInfo`, `useAllNfts`, `useCollectionNfts`, `useNftDetail`, `useTransferHistory`, `useFavorites`, `useGalleryFilters`, `useEthPrice`, `useRecentlyViewed`
- **useRef + IntersectionObserver** — infinite scroll для автоподгрузки NFT

### Сравнение с аналогами

| Критерий | React | Vue.js | Angular | Svelte |
|---|---|---|---|---|
| Подход | Библиотека UI | Прогрессивный фреймворк | Полный фреймворк | Компилятор |
| Кривая обучения | Средняя | Низкая | Высокая | Низкая |
| Размер бандла | Средний | Маленький | Большой | Минимальный |
| Экосистема | Огромная | Большая | Большая | Растущая |
| Типизация | Через TypeScript | Через TypeScript | TypeScript встроен | Через TypeScript |

React выбран за огромную экосистему, зрелость и отличную совместимость с Web3-библиотеками.

## Vite

### Общее описание
Vite — инструмент сборки нового поколения. Использует нативные ES-модули для мгновенного dev-сервера и Rollup для production-сборки.

### Зачем использовался
- **Мгновенный запуск** dev-сервера (< 300мс)
- **HMR** — изменения отображаются за миллисекунды
- **Оптимизированная сборка** — tree-shaking, code splitting, минификация
- **Поддержка TypeScript, JSX, CSS** из коробки
- **Алиасы путей** (`@/` → `src/`)
- **Интеграция с Vitest** — единый конфиг для сборки и тестов

### Сравнение с аналогами

| Критерий | Vite | Webpack | Create React App | Turbopack |
|---|---|---|---|---|
| Скорость dev-сервера | Мгновенная | Медленная | Медленная | Быстрая |
| Конфигурация | Минимальная | Сложная | Нулевая (ограниченная) | Минимальная |
| HMR | < 50мс | 100-500мс | 100-500мс | < 50мс |
| Production сборка | Rollup | Webpack | Webpack | В разработке |

Vite выбран как современная и быстрая альтернатива Webpack с минимальной конфигурацией.

## Tailwind CSS

### Зачем использовался
Tailwind обеспечивает весь визуальный слой:

- **Кастомная тема** — палитра NFT-цветов (`nft-dark`, `nft-violet`, `nft-pink`, `nft-cyan`, `nft-lime`), анимации (`float`, `glow`, `slide-up`, `fade-in`)

- **Тёмная/светлая тема** — через условные классы и `isDark` из контекста
- **Glassmorphism** — `backdrop-blur`, полупрозрачные фоны, градиенты
- **Утилитарные компоненты** — `btn-primary`, `gradient-text`, `glass-card` через `@apply`

### Сравнение с аналогами

| Критерий | Tailwind CSS | Bootstrap | MUI | Styled Components |
|---|---|---|---|---|
| Подход | Utility-first | Компонентный | Компонентный | CSS-in-JS |

## Web3.js

### Общее описание
Web3.js — JavaScript-библиотека для взаимодействия с блокчейном Ethereum. Предоставляет API для чтения данных, отправки транзакций и работы со смарт-контрактами.

### Зачем использовался
Web3.js — ключевая технология проекта. Всё, что видит пользователь, приходит из блокчейна:

- **Публичный RPC fallback** — приложение работает без MetaMask через `ethereum-rpc.publicnode.com`
- **Чтение ERC-721 контрактов** — `name()`, `symbol()`, `totalSupply()`, `tokenURI()`, `ownerOf()` для каждого NFT
- **Резолв метаданных** — `tokenURI` возвращает IPFS/Arweave/HTTP ссылку на JSON с метаданными; сервис резолвит `ipfs://` через публичные шлюзы, парсит `data:application/json;base64,...`, скачивает и кэширует
- **Автоопределение нумерации** — `detectStartId()` проверяет, начинаются ли токены с 0 или с 1
- **История трансферов** — чтение `Transfer` событий через `getPastLogs` с декодированием topics
- **Подключение MetaMask** — `eth_requestAccounts`, подписка на `accountsChanged`/`chainChanged`
- **Данные блокчейна** — баланс, номер блока, gas price, количество транзакций адреса
- **Динамическая валидация** — при добавлении контракта в каталог вызывается `name()` для проверки, что это ERC-721

### Сравнение с аналогами

| Критерий | Web3.js | Ethers.js | Viem | wagmi |
|---|---|---|---|---|
| Размер | Большой (~750KB) | Средний (~120KB) | Маленький (~35KB) | Средний + Viem |
| API | ООП (классы) | Функциональный + ООП | Функциональный | React хуки |
| Типизация | TypeScript | TypeScript | TypeScript (строгая) | TypeScript |
| Зрелость | Очень высокая (2015+) | Высокая (2016+) | Средняя (2023+) | Средняя (2022+) |
| Поддержка | Ethereum Foundation | Сообщество | Сообщество | Сообщество |

Web3.js выбран как самая зрелая библиотека, поддерживаемая Ethereum Foundation, с обширной документацией.

Vitest — фреймворк тестирования для экосистемы Vite. React Testing Library — библиотека для тестирования компонентов через поведение.

### Зачем использовались

### Сравнение с аналогами

|---|---|---|---|---|
| Скорость | Очень высокая | Средняя | Средняя | Низкая |
| Конфигурация | Общая с Vite | Отдельная | Сложная | Отдельная |
| ESM-поддержка | Нативная | Через трансформацию | Нативная | Нативная |

## GitHub Actions (CI/CD)

### Общее описание
GitHub Actions — встроенная платформа CI/CD для GitHub. Автоматизирует сборку, тестирование и деплой.

### Зачем использовался
Пайплайн из трёх задач:

1. **Lint** — ESLint

## Заключение

### Рефлексия по проделанной работе

В ходе разработки NFT Gallery удалось создать полноценное веб-приложение с on-chain интеграцией, без единого мока:

1. **Архитектура** — Feature Sliced Design с чётким разделением: типы → конфигурация → сервисы (Web3, NFT) → хуки → контексты → компоненты → страницы. Реактивный `ContractStore` с подпиской через `useSyncExternalStore` для динамического каталога контрактов.

2. **On-chain данные** — весь контент приложения загружается из блокчейна: `name()`, `symbol()`, `totalSupply()` для коллекций; `tokenURI()` + резолв IPFS/Arweave + парсинг JSON для метаданных NFT; `ownerOf()` для владельцев; `getPastLogs` для истории трансферов. Публичный RPC fallback позволяет работать без MetaMask.

3. **UI/UX** — тёмная тема в стиле NFT-маркетплейсов с glassmorphism, градиентами и анимациями. Infinite scroll через IntersectionObserver. Skeleton-анимации при загрузке. Динамическое добавление контрактов с on-chain валидацией. Полностью адаптивный mobile-first дизайн.

4. **Оптимизация** — двухуровневое кэширование метаданных (memory + localStorage с TTL), мемоизация компонентов, дебаунс поиска, ленивая загрузка страниц, пакетная загрузка NFT.

5. **Тестирование** — 62 теста на трёх уровнях. Покрыты ключевые сценарии: фильтрация, поиск, избранное, переключение темы, рендеринг on-chain данных.

6. **CI/CD** — GitHub Actions: lint → test → build, автоматический запуск на каждый push и PR.

### Впечатления от технологий

- **React 19** — стабильная и зрелая библиотека. `useSyncExternalStore` — удобный способ подписки на внешние store без контекстов. Custom hooks позволяют красиво инкапсулировать сложную логику загрузки blockchain-данных.
- **Vite** — значительно ускоряет разработку. Мгновенный HMR и минимальная конфигурация.
- **Tailwind CSS** — после освоения значительно ускоряет вёрстку. Позволяет создавать уникальные дизайны без написания CSS-файлов. Длинные строки классов — единственный минус.
- **Web3.js** — мощная, но тяжёлая библиотека. Работа с публичным RPC и fallback-стратегия требует аккуратной обработки ошибок. Для нового проекта стоит рассмотреть Viem (легче, строже типизирован), но Web3.js наиболее документирован.
- **Vitest** — идеальный компаньон для Vite. Единый конфиг, мгновенный watch, совместимость с Jest API.
- **GitHub Actions** — максимально простая настройка CI. YAML-конфиг понятен, marketplace экономит время.
