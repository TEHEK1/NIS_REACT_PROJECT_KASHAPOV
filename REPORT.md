# NFT Gallery — Отчёт

## Суть приложения

**NFT Gallery** — это веб-приложение для просмотра и взаимодействия с коллекциями NFT (невзаимозаменяемых токенов). Приложение представляет собой одностраничное SPA с галереей цифрового искусства, где пользователь может:

- Просматривать NFT в виде сетки карточек с изображениями, ценами и информацией о создателях
- Фильтровать и сортировать NFT по коллекции, ценовому диапазону, типу (аукцион/фиксированная цена)
- Просматривать детальную информацию о каждом NFT: описание, свойства с показателями редкости, создатель, владелец
- Изучать коллекции со статистикой (floor price, объём торгов, количество элементов)
- Подключать криптокошелёк MetaMask и взаимодействовать с блокчейном Ethereum: просмотр баланса, сети, последнего блока и gas price
- Сохранять понравившиеся NFT в избранное
- Переключаться между тёмной и светлой темами

Приложение демонстрирует применение современных Frontend-технологий в контексте Web3/блокчейн-домена.

## React

### Общее описание
React — это JavaScript-библиотека для построения пользовательских интерфейсов, разработанная компанией Meta (Facebook). React использует декларативный подход, компонентную архитектуру и виртуальный DOM для эффективного обновления UI.

### Зачем использовался
React является основой приложения. Все UI-элементы реализованы как React-компоненты с использованием хуков (`useState`, `useEffect`, `useMemo`, `useCallback`, `useContext`, `memo`). Основные паттерны:

- **React Context** — для глобального состояния (тема, кошелёк) без prop drilling
- **React.lazy + Suspense** — ленивая загрузка страниц для оптимизации бандла
- **memo** — мемоизация компонентов для предотвращения лишних перерисовок
- **Custom hooks** — инкапсуляция логики (useFavorites, useGalleryFilters, useWallet, useTheme)

### Сравнение с аналогами

| Критерий | React | Vue.js | Angular | Svelte |
|---|---|---|---|---|
| Подход | Библиотека UI | Прогрессивный фреймворк | Полный фреймворк | Компилятор |
| Кривая обучения | Средняя | Низкая | Высокая | Низкая |
| Размер бандла | Средний | Маленький | Большой | Минимальный |
| Экосистема | Огромная | Большая | Большая | Растущая |
| Типизация | Через TypeScript | Через TypeScript | TypeScript встроен | Через TypeScript |

React был выбран за огромную экосистему, зрелость, широкую поддержку сообщества и отличную совместимость с Web3-библиотеками.

## Vite

### Общее описание
Vite — это инструмент сборки нового поколения для JavaScript-приложений. Он использует нативные ES-модули браузера для молниеносного dev-сервера и Rollup для оптимизированной production-сборки.

### Зачем использовался
Vite обеспечивает:
- **Мгновенный запуск** dev-сервера (< 300мс) — благодаря нативным ESM, не нужно пересобирать весь бандл
- **Горячая замена модулей (HMR)** — изменения в коде отображаются в браузере за миллисекунды
- **Оптимизированная сборка** — tree-shaking, code splitting, минификация через Rollup
- **Поддержка TypeScript, JSX, CSS** из коробки без дополнительной конфигурации
- **Алиасы путей** (`@/` → `src/`) для удобного импорта

### Сравнение с аналогами

| Критерий | Vite | Webpack | Create React App | Turbopack |
|---|---|---|---|---|
| Скорость dev-сервера | Мгновенная | Медленная | Медленная | Быстрая |
| Конфигурация | Минимальная | Сложная | Нулевая (ограниченная) | Минимальная |
| HMR | < 50мс | 100-500мс | 100-500мс | < 50мс |
| Production сборка | Rollup | Webpack | Webpack | В разработке |
| Поддержка плагинов | Rollup-совместимые | Огромная | Ограниченная | Ограниченная |

Vite был выбран как современная и быстрая альтернатива Webpack, с минимальной конфигурацией и отличной поддержкой React + TypeScript.

## Tailwind CSS

### Общее описание
Tailwind CSS — это utility-first CSS-фреймворк, предоставляющий набор низкоуровневых утилитарных классов для построения пользовательских дизайнов прямо в разметке. Вместо написания CSS-файлов, стили задаются через классы (`flex`, `p-4`, `text-white`, `hover:bg-blue-500`).

### Зачем использовался
Tailwind CSS обеспечивает весь визуальный слой приложения:

- **Кастомная тема** — расширенная палитра цветов (`nft-dark`, `nft-violet`, `nft-pink`, `nft-cyan`), кастомные анимации (`float`, `glow`, `slide-up`, `fade-in`)
- **Responsive дизайн** — mobile-first с брейкпоинтами `sm:`, `md:`, `lg:`, `xl:`
- **Тёмная/светлая тема** — через класс `dark` на `<html>` и условные классы в компонентах
- **Компонентные утилиты** — `glass`, `glass-card`, `gradient-text`, `btn-primary`, `btn-secondary` через директиву `@apply`
- **Современные эффекты** — backdrop-blur (glassmorphism), градиенты, тени, анимации переходов

### Сравнение с аналогами

| Критерий | Tailwind CSS | Bootstrap | MUI (Material UI) | Styled Components |
|---|---|---|---|---|
| Подход | Utility-first | Компонентный | Компонентный | CSS-in-JS |
| Кастомизация | Полная | Средняя | Средняя | Полная |
| Размер бандла | Минимальный (PurgeCSS) | Средний | Большой | Зависит от использования |
| Дизайн-система | Своя | Material-like | Material Design | Своя |
| Переиспользование | Через @apply | Готовые компоненты | Готовые компоненты | Компоненты |
| Кривая обучения | Средняя | Низкая | Средняя | Средняя |

Tailwind был выбран для максимальной гибкости дизайна — в отличие от компонентных фреймворков (Bootstrap, MUI), он не навязывает готовый визуальный стиль, позволяя создать уникальный интерфейс для NFT-галереи.

## Web3.js

### Общее описание
Web3.js — это JavaScript-библиотека для взаимодействия с блокчейном Ethereum. Она предоставляет API для подключения к узлам Ethereum, чтения данных из блокчейна, отправки транзакций и взаимодействия со смарт-контрактами.

### Зачем использовался
Web3.js является ключевой технологией для Web3-интеграции:

- **Подключение кошелька** — соединение с MetaMask через `window.ethereum` провайдер
- **Чтение данных блокчейна** — получение баланса ETH, номера блока, цены газа, ID сети
- **ERC-721 ABI** — определение интерфейса для взаимодействия с NFT-контрактами (ownerOf, tokenURI, balanceOf)
- **Подписка на события** — отслеживание смены аккаунта и сети в реальном времени
- **Конвертация единиц** — перевод Wei в ETH и обратно через `web3.utils`

Сервис `Web3Service` инкапсулирует всю работу с блокчейном и предоставляет удобный TypeScript API для использования в React-компонентах через `WalletContext`.

### Сравнение с аналогами

| Критерий | Web3.js | Ethers.js | Viem | wagmi |
|---|---|---|---|---|
| Размер | Большой (~750KB) | Средний (~120KB) | Маленький (~35KB) | Средний + Viem |
| API | ООП (классы) | Функциональный + ООП | Функциональный | React хуки |
| Типизация | TypeScript | TypeScript | TypeScript (строгая) | TypeScript |
| Зрелость | Очень высокая (2015+) | Высокая (2016+) | Средняя (2023+) | Средняя (2022+) |
| Поддержка | Ethereum Foundation | Сообщество | Сообщество | Сообщество |

Web3.js был выбран как самая зрелая и документированная библиотека для работы с Ethereum, поддерживаемая Ethereum Foundation с обширной базой знаний и примеров.

## Vitest + React Testing Library

### Общее описание
Vitest — это молниеносный фреймворк для тестирования, созданный специально для экосистемы Vite. Он использует тот же конфиг и пайплайн трансформации, что и Vite, обеспечивая идентичную среду для тестов и разработки. React Testing Library — библиотека для тестирования React-компонентов через их поведение, а не внутреннюю реализацию.

### Зачем использовались
Тестирование покрывает три уровня приложения (66 тестов в 10 файлах):

- **Unit-тесты утилит** (22 теста) — `formatEth`, `truncateAddress`, `timeAgo`, `generateGradient` и другие чистые функции
- **Тесты хуков** (18 тестов) — `useFavorites` (добавление, удаление, localStorage-персистенция), `useGalleryFilters` (поиск, фильтрация по коллекции/цене/аукционам, сортировка, комбинация фильтров, сброс)
- **Компонентные тесты** (26 тестов) — `NFTCard` (рендеринг данных, ссылка, избранное, аукцион), `SearchBar` (debounce, очистка), `ThemeToggle` (переключение темы), `NFTGrid` (пустое состояние), `TraitBadge` (уровни редкости), `StatsCard`

Ключевые паттерны: `renderHook` для изолированного тестирования хуков, `renderWithProviders` (обёртка с ThemeProvider + MemoryRouter), `fireEvent` для управления fake timers при debounce, `userEvent` для реалистичной симуляции пользователя.

### Сравнение с аналогами

| Критерий | Vitest | Jest | Mocha | Cypress Component |
|---|---|---|---|---|
| Скорость | Очень высокая (Vite HMR) | Средняя | Средняя | Низкая |
| Конфигурация | Общая с Vite | Отдельная | Сложная | Отдельная |
| ESM-поддержка | Нативная | Через трансформацию | Нативная | Нативная |
| Watch-режим | Мгновенный | Средний | Ручной | Средний |
| Совместимость с Jest API | Полная | — | Нет | Нет |

Vitest выбран как нативный компаньон Vite — единый конфиг, единая система трансформации модулей, мгновенный watch-режим. Jest API полностью совместим, поэтому миграция тривиальна.

## GitHub Actions (CI/CD)

### Общее описание
GitHub Actions — встроенная в GitHub платформа для непрерывной интеграции и доставки (CI/CD). Позволяет автоматизировать сборку, тестирование и деплой при каждом push или pull request.

### Зачем использовался
CI-пайплайн состоит из трёх последовательных задач:

1. **Lint** — запуск ESLint для проверки стиля и потенциальных ошибок
2. **Test** — запуск всех 66 тестов через Vitest
3. **Build** — production-сборка через `tsc + vite build` (запускается только после успешных lint и test)

Пайплайн активируется на каждый `push` и `pull_request` в ветки main/master. Артефакт сборки (`dist/`) сохраняется на 7 дней.

### Сравнение с аналогами

| Критерий | GitHub Actions | GitLab CI | Jenkins | CircleCI |
|---|---|---|---|---|
| Интеграция с GitHub | Нативная | Нет | Через плагин | Через интеграцию |
| Конфигурация | YAML | YAML | Groovy/UI | YAML |
| Бесплатный план | 2000 мин/мес | 400 мин/мес | Self-hosted | 6000 мин/мес |
| Marketplace | Огромный | Средний | Огромный (плагины) | Средний |
| Настройка | Простая | Простая | Сложная | Средняя |

GitHub Actions выбран как нативное CI-решение для GitHub-репозитория — нулевая настройка интеграции, огромный marketplace готовых actions, щедрый бесплатный план.

## React Router

### Общее описание
React Router — библиотека для клиентской маршрутизации в React-приложениях. Позволяет создавать SPA с несколькими «страницами» без перезагрузки.

### Зачем использовался
React Router обеспечивает навигацию между страницами:
- **Ленивая загрузка** — каждая страница загружается только при переходе (code splitting)
- **Вложенные маршруты** — общий Layout с Header и Footer оборачивает все страницы
- **Динамические параметры** — `/nft/:id` и `/collections/:slug` для детальных страниц
- **Активные ссылки** — `NavLink` с визуальным выделением текущей страницы

### Сравнение с аналогами

| Критерий | React Router | TanStack Router | Next.js Router |
|---|---|---|---|
| Тип | SPA | SPA | SSR/SSG + SPA |
| Типизация маршрутов | Слабая | Строгая | Файловая система |
| Ленивая загрузка | Ручная (React.lazy) | Встроенная | Автоматическая |
| Зрелость | Очень высокая | Средняя | Высокая |

## Заключение

### Рефлексия по проделанной работе

В ходе разработки NFT Gallery удалось создать полноценное веб-приложение с современным UI и реальной интеграцией с блокчейном. Основные достижения:

1. **Архитектура** — чистое разделение на слои: типы → данные → сервисы → контексты → компоненты → страницы. React Context для глобального состояния вместо тяжёлых state-менеджеров.

2. **UI/UX** — тёмная тема в стиле NFT-маркетплейсов с glassmorphism-эффектами, градиентами и анимациями. Полностью адаптивный дизайн с mobile-first подходом.

3. **Web3 интеграция** — полноценный Web3Service с поддержкой подключения MetaMask, чтения данных блокчейна, подписки на события и ERC-721 ABI для NFT-контрактов.

4. **Оптимизация** — ленивая загрузка страниц, мемоизация компонентов, дебаунс поиска, кастомные хуки для переиспользования логики.

5. **Тестирование** — 66 тестов на трёх уровнях: unit-тесты утилит, тесты хуков через renderHook, компонентные тесты через Testing Library. Покрыты ключевые сценарии: фильтрация, поиск, избранное, переключение темы.

6. **CI/CD** — GitHub Actions пайплайн с тремя этапами (lint → test → build), автоматический запуск на каждый push и PR.

### Впечатления от технологий

- **React 19** — стабильная и зрелая библиотека. Хуки позволяют писать чистый и понятный код. Context API достаточен для приложений среднего размера.
- **Vite** — значительно ускоряет разработку по сравнению с Webpack. Минимальная конфигурация и мгновенный HMR.
- **Tailwind CSS** — на первых порах непривычен из-за длинных строк классов, но после освоения значительно ускоряет вёрстку и позволяет создавать уникальные дизайны без написания CSS.
- **Web3.js** — мощная, но тяжёлая библиотека. Для простых задач подошёл бы более лёгкий аналог (Ethers.js или Viem). Однако Web3.js наиболее документирован и поддерживается Ethereum Foundation.
- **Vitest** — идеальный компаньон для Vite. Единый конфиг, мгновенный watch, полная совместимость с Jest API — переход с Jest практически бесшовный.
- **GitHub Actions** — максимально простая настройка CI для GitHub-репозитория. YAML-конфиг понятен, marketplace actions экономит время.
